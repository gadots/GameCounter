# design-sync notes — GameCounter

Repo-specific gotchas for future syncs. Read before re-running.

## Shape & entry
- This is a **Vite app**, not a component library — no library build, no `dist/` entry,
  no shipped `.d.ts`. The DS is a hand-picked subset of the app's components.
- We use an explicit **barrel entry** `.design-sync/entry.tsx` (passed via `--entry`)
  that exports only the 7 DS components + `MemoryRouter`. Do NOT fall back to the
  converter's synth-entry (`export * from` every src file) — it would bundle
  `src/main.tsx`, whose top-level `createRoot().render()` runs on bundle load and
  breaks every preview.
- The 7 components are pinned in `cfg.componentSrcMap`. **Adding a new DS component
  requires editing BOTH `entry.tsx` (add the export) AND `componentSrcMap`** —
  discovery is pinned, not automatic.

## CSS (Tailwind v4)
- No static stylesheet ships with the app. `cfg.buildCmd` compiles a stable stylesheet
  with `@tailwindcss/cli@4` → `.design-sync/tw/compiled.css` (the `cssEntry`).
- `compiled.css` is **gitignored** — it MUST be regenerated by running `buildCmd`
  before the converter on every sync, or the CSS goes empty/stale.
- `tw/input.css` `@source` globs scan `src/components`, `src/pages`, and
  `.design-sync/previews`. **Any Tailwind class used only in an authored preview must
  be under one of those globs** or it won't be in the compiled CSS.

## Providers & fixed-position components
- `provider = MemoryRouter` (re-exported from the barrel). `PageHeader` and `BottomNav`
  need react-router context; the others don't.
- `Modal` and `BottomNav` are `position: fixed`. Their preview `.tsx` wraps them in a
  `transform: translateZ(0)` containing block so the fixed element anchors to the card
  instead of the viewport. Both use `cfg.overrides.<Name>: {cardMode: "single", viewport}`.
  If either renders blank on re-sync, that wrapper is why — keep it.
- `useTranslation` is a `localStorage`/`useSyncExternalStore` store (no context);
  defaults to Spanish. Previews assume `es` copy.

## Environment
- npm lockfile has drift (playwright 1.52 in lock vs 1.56 installed) → `npm ci` FAILS.
  Use `npm install`.
- Chromium is pre-installed at `/opt/pw-browsers` (build 1194) and matches
  playwright-core 1.56, so the render check runs with no extra install.

## Groups
- Grouping is `general` (5 ui components) + `layout` (2). "general" is the generic
  fallback because `src/components/ui` is a generic dir name. To refine without losing
  the synthesized `.prompt.md`, add a JSDoc `@category <Group>` to each `ui/` component
  source (not done — kept app source untouched for the first sync).

## Re-sync risks (watch-list)
- **compiled.css is gitignored** — always run `buildCmd` first, else empty CSS.
- **New DS components** need entry.tsx + componentSrcMap edits (pinned discovery).
- **Preview-only Tailwind classes** must fall under `tw/input.css` `@source` globs.
- Grades in `.design-sync/.cache/` are gitignored — cross-machine verified-state comes
  from the uploaded `_ds_sync.json`; a fresh clone re-verifies from scratch.
